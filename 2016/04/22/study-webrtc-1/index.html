# 研究 WebRTC &1 初次认识

Published: 2016-04-22

关键词：Google I/O 16 / 初步认识 WebRTC / Duo / 学习资料

这次的 Google 2016 开发者大会上，有一款视频通话的产品叫做 Duo，同样让人眼前一亮，对它的介绍是这样的：“ Duo 的最大亮点就是用户在决定接通之前可以提前在视频中看到对方，借此来决定是否接通这个视频。Duo 基于 Google 全新的 QUIC 和 WebRTC，WebRTC 是网页实时通信（Web Real-Time Communication）的缩写，支持网页浏览器进行实时语音对话或视频对话的技术，6 年前 Google 通过收购获得这项技术。QUIC(Quick UDP Internet Connection) 是 Google 制定的一种基于 UDP 的低延时的互联网传输层协议，也是 Chrome 为什么这么快的原因之一。比起其它视频聊天软件，Duo 对数据需求更小，视频更稳定清晰 ”。

我觉得，WebRTC 有 Google 和万维网联盟的支持与推动一定会大放异彩。整理了一下其相关的资料，了解了解 WebRTC 的前世今生。

1. 认识 WebRTC #

WebRTC 是一个支持网页浏览器进行实时语音对话或视频对话的 API。WebRTC 使用 GIPS 引擎，实现了基于网页的视频会议，并支持 722，PCM，ILBC，ISAC 等编码，同时使用谷歌自家的 VP8 视频解码器；同时支持 RTP/SRTP 传输等。2012 年 1 月，谷歌已经把这款软件集成到 Chrome 浏览器中。同时 FreeSWITCH 项目宣称支持 iSAC audio codec。

说起 WebRTC 不得不提 GIPS 公司，全称 Global IP Solutions。是一家位于瑞典斯德哥尔摩的专门从事处理 IP 语音技术的高科技公司，也就是 IP 语音引擎和相关产品。于 1999 年成立，后改名为 Global IP Solutions。许多大公司如，北电网络、Skype、QQ、WebEx、AOL、EarthLink 等目前都采用其公司的产品 GIPS SoundWare™。

而 Google 在 2010 年收购了 GIPS（Global IP Solutions）公司，从而获得 WebRTC 技术，并在 2011 年，按 BSD 协议把 WebRTC 开源了，同年 w3c 启动 WebRTC 计划，使 WebRTC 成为 HTML5 标准的一部分，目前该规范还在开发中。

2011 年以前，浏览器之间要实现实时通信需要私有技术，这些技术大部分都是通过插件和客户端来安装使用。对于许多用户来说，插件的下载、安装和更新是一个复杂、繁琐和容易出错的操作。对于开发人员来说，插件的的调试、测试、部署、错误修复和维护同样很难，同时涉及到的一些技术是受版权保护的，整合也是很复杂的。另外，有时很难说服用户来安装插件。

Google 通过把 WebRTC 项目开源，希望浏览器厂商能够将该技术内建在浏览器中，从而使 Web 应用开发人员能够通过 HTML 标签和 JavaScript API 就实现 Web 音频、视频通信功能。Google 在其官网上列出了使用 WebRTC 技术的四个理由：

  • 互联网成功的一个关键因素是一些核心技术如 HTML、HTTP 和 TCP/IP，而且是开放和免费实现的。目前，在浏览器通信领域还没有免费、高质量、完整的解决方案，而 WebRTC 就是这样的技术。
  • 该技术的使用已经超过了 8 年，集成了最佳的音频、视频引擎，并被部署到数以百万的终端中。
  • 该技术包含了使用 STUN、ICE、TURN、RTP-over-TCP 的关键 NAT 和防火墙穿透技术，并支持代理。
  • 通过浏览器，WebRTC 把通讯双方的信令状态直接映射到 PeerConnection 里面来抽象信令处理，这样，开发人员按不同的应用场景选择不同的会话协议，比如 SIP、XMPP/Jingle 等等）。

Chrome 从版本 20 开始支持 WebRTC，实现主要参考 chromium 里的实现进行移植。
WebKit 库方面，除了适配，已经支持 WebRTC，操作如下：

  • 编译时启用宏 ENABLE_MEDIA_STREAM；
  • 采用 http://svn.webkit.org/repository/webkit/trunk/Source/WebCore/Modules/mediastream 下面的所以 idl 文件，并把生成的 cpp 文件添加到 WebCore工程里；
  • 把 http://svn.webkit.org/repository/webkit/trunk/Source/WebCore/Modules/mediastream 下面的所有 cpp 文件添加到 WebCore工程里；
  • 把 http://svn.webkit.org/repository/webkit/trunk/Source/WebCore/platform/mediastream 下面的所有 cpp 文件添加到 WebCore工程里；
